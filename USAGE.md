# Руководство по использованию Flash Cards

## Быстрый старт

### Для Linux/MacOS:

```bash
chmod +x start.sh
./start.sh
```

### Для Windows:

```bash
docker compose up --build
```

Откройте браузер: http://localhost:8000

## Основные функции

### 1. Добавление слов

1. Перейдите на страницу "Управление словами"
2. Нажмите кнопку "Добавить слово"
3. Выберите язык (английский или испанский)
4. Введите иностранное слово и нажмите "Поиск"
5. Выберите один из предложенных переводов или введите свой
6. Нажмите "Добавить слово"

Слова загружаются из справочника базовых слов в базе данных.
Вы можете выбрать слова из готового словаря или добавить свои собственные переводы.

### 2. Управление словами

На странице "Мои слова" вы можете:

- Просмотреть все добавленные слова
- Фильтровать по языку
- Фильтровать по статусу (выучено/не выучено)
- Отмечать слова как выученные
- Удалять слова

### 3. Тренировки

#### Этап 1: Просмотр и запоминание

- Показывается иностранное слово
- Автоматическая озвучка (работает в браузерах с поддержкой Web Speech API)
- Нажмите на карточку, чтобы увидеть перевод
- Посмотрите примеры использования
- Переходите к следующему слову

#### Этап 2: Выбор перевода

- Показывается иностранное слово
- Выберите правильный перевод из 4 вариантов
- Система автоматически покажет результат
- Статистика правильных ответов отображается вверху

#### Этап 3: Сопоставление

- Сопоставьте иностранные слова с их переводами
- Кликайте на слова слева и справа для создания пар
- Тренировка идёт группами по 4 слова
- При правильном совпадении пара исчезает

#### Этап 4: Составление слова

- Показывается русский перевод
- Составьте иностранное слово из перемешанных букв
- Кликайте на буквы для добавления в слово
- Кликайте на составленное слово для удаления букв
- Проверьте результат

### 4. Фильтрация

Вы можете фильтровать слова по:

- **Языку**: Все языки, Английский, Испанский
- **Статусу**: Все, Не выучено, Выучено

## API Endpoints

### Слова

```bash
# Получить все слова
GET /api/words?language=ENGLISH&status=NOT_LEARNED

# Получить одно слово
GET /api/words/:id

# Добавить слово
POST /api/words
Body: {
  "foreignWord": "hello",
  "translation": "привет",
  "language": "ENGLISH",
  "examples": ["Hello, world!"]
}

# Обновить слово
PATCH /api/words/:id
Body: { "status": "LEARNED" }

# Удалить слово
DELETE /api/words/:id
```

### Тренировки

```bash
# Записать результат тренировки
POST /api/training
Body: {
  "wordId": "uuid",
  "stage": 1,
  "isCorrect": true
}

# Получить статистику
GET /api/training?wordId=uuid
```

## База данных

### Prisma Studio

Для просмотра и управления данными в БД:

```bash
# Если запущено через Docker
docker compose exec app npx prisma studio

# Если запущено локально
npx prisma studio
```

Откроется интерфейс на http://localhost:5555 (порт Prisma Studio по умолчанию)

### Миграции

```bash
# Создать новую миграцию
docker compose exec app npx prisma migrate dev --name migration_name

# Применить миграции
docker compose exec app npx prisma migrate deploy
```

## Работа с переводами

Система хранит переводы слов в таблице `WordTranslation` в базе данных.
Переводы загружаются из справочника базовых слов при добавлении слова через диалог.

```bash
# Остановить и удалить все контейнеры и volumes
docker compose down -v

# Запустить заново
docker compose up --build
```

## Расширение функционала

### Добавление новых слов в справочник

Слова добавляются через seed-скрипт в базу данных:

```bash
# Запустите seed для загрузки базовых слов
docker compose exec app npx prisma db seed
```

### Добавление поддержки новых языков

1. Добавьте язык в таблицу `Language` через Prisma Studio или миграцию
2. Создайте части речи для нового языка в таблице `PartOfSpeech`
3. Добавьте базовые слова в таблицу `BaseWord` с переводами в `WordTranslation`

## Производительность

- Переводы хранятся в БД для быстрого доступа
- NextJS автоматически кеширует статические страницы
- Prisma использует connection pooling

## Поддержка браузеров

- Озвучка слов работает в Chrome, Edge, Safari (Web Speech API)
- Все современные браузеры поддерживают остальной функционал

## Troubleshooting

### Порт 8000 уже занят

```bash
# Измените порт в docker-compose.yml
ports:
  - "8001:8000"
```

### Порт 5433 уже занят

```bash
# Измените порт PostgreSQL в docker-compose.yml
ports:
  - "5434:5432"

# Не забудьте обновить DATABASE_URL
```

### Ошибка подключения к БД

```bash
# Проверьте, что PostgreSQL запущен
docker compose ps

# Перезапустите сервисы
docker compose restart
```
