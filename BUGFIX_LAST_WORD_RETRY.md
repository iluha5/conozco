# Исправление критического бага: Ошибка на последнем слове

## 🐛 Проблема

**Корнер-кейс:** Если на последнем этапе тренировки все слова сделаны правильно, кроме последнего, и в последнем слове допустить ошибку, программа не переходила в режим повтора (retry mode).

**Воспроизведение:**

1. Дойти до последнего слова в этапе
2. Все предыдущие слова выполнены правильно
3. Допустить ошибку в последнем слове
4. **Результат:** Программа "застревает", не переходит в retry mode

## 🔍 Причина

В компонентах Stage2, Stage4, Stage5, Stage6 была неполная логика обработки неправильных ответов:

### Было (баг):

```typescript
if (correct) {
    // Автоматический переход для правильных ответов
    setTimeout(() => {
        if (isRetryMode) {
            /* retry логика */
        } else {
            handleNext();
        }
    }, 1000);
} else if (isRetryMode && !correct) {
    // Обработка ошибок ТОЛЬКО в retry mode
    setTimeout(() => {
        /* retry логика */
    }, 2000);
}
// ❌ ОТСУТСТВУЕТ обработка для (!isRetryMode && !correct)
```

**Проблема:** Отсутствовал блок `else` для обработки неправильных ответов в обычном режиме (не retry mode).

### Стало (исправлено):

```typescript
const delay = correct ? 1000 : 2000;
setTimeout(() => {
    if (isRetryMode) {
        if (correct) {
            /* успешное исправление */
        } else {
            /* повторная ошибка */
        }
    } else {
        // ✅ ВСЕГДА вызывается для любого ответа в обычном режиме
        handleNext();
    }
}, delay);
```

**Решение:** Добавлен блок `else`, который ВСЕГДА вызывает `handleNext()` в обычном режиме, независимо от правильности ответа.

## 🎯 Как работает исправление

### Обычный режим (первый проход):

1. **Правильный ответ на последнем слове:**

    ```
    correct=true → delay=1000мс → handleNext()
    → currentIndex === lastIndex
    → errorIndices.length === 0
    → onComplete() ✅
    ```

2. **Неправильный ответ на последнем слове:**
    ```
    correct=false → delay=2000мс → handleNext()
    → currentIndex === lastIndex
    → errorIndices.length > 0
    → setIsRetryMode(true)
    → setCurrentIndex(firstErrorIndex) ✅
    ```

### Retry режим:

3. **Исправил ошибку:**

    ```
    isRetryMode=true, correct=true
    → findNextError() === -1
    → onComplete() ✅
    ```

4. **Снова ошибся:**
    ```
    isRetryMode=true, correct=false
    → findNextError() → переход к следующей ошибке
    → или перезагрузка карточки если это единственная ошибка ✅
    ```

### Stage 3 (особый случай)

Stage3 использует другую архитектуру с батчами (по 10 слов). Переход происходит автоматически после завершения батча через `useEffect` с зависимостью от `allMatched`.

Логика проверена и работает правильно:

```typescript
useEffect(() => {
    if (allMatched && pairs.length > 0) {
        // ... проверка последнего батча
        const hasAnyErrors = exerciseResults.some(r => r === false);
        if (hasAnyErrors) {
            setIsRetryMode(true);
            setCurrentBatch(firstErrorBatch);
        }
    }
}, [allMatched, ...]);
```

### Таймауты

- **Правильный ответ:** 1000мс (1 секунда)
- **Неправильный ответ:** 2000мс (2 секунды)

Это дает пользователю время увидеть результат перед автоматическим переходом.
