# План внедрения E2E тестов на Playwright

## Требования к браузерам

**Используется только Chromium браузер.**

- Firefox и WebKit **не используются** для ускорения тестов и уменьшения размера зависимостей
- Установка только Chromium: `npx playwright install chromium`
- В конфигурации Playwright указывается только `chromium` в списке проектов

## Обзор

Внедрение комплексной системы end-to-end тестирования для приложения Flash Cards с использованием Playwright. Тесты будут покрывать все основные пользовательские сценарии: авторизацию, управление словами, тренировки и настройки.

**Важно:** Для ускорения тестов и уменьшения размера зависимостей используется только браузер Chromium (без Firefox и WebKit).

## Этапы реализации

### 1. Установка и настройка Playwright

**Файлы:**

- `package.json` - добавление зависимостей
- `playwright.config.ts` - конфигурация Playwright
- `.github/workflows/e2e.yml` или `.gitlab-ci.yml` - CI/CD конфигурация

**Действия:**

- Установить `@playwright/test` как dev зависимость
- Установить **только Chromium браузер**: `npx playwright install chromium` (Firefox и WebKit не устанавливаются)
- Настроить конфигурацию `playwright.config.ts` с одним проектом для Chromium
- Настроить конфигурацию для локальной разработки и CI
- Настроить параллельный запуск тестов
- Добавить скриншоты и видео при падении тестов

### 2. Настройка тестовой среды с Docker

**Файлы:**

- `docker-compose.test.yml` - отдельный compose файл для тестов
- `e2e/global-setup.ts` - запуск тестовой БД перед тестами
- `e2e/global-teardown.ts` - очистка после тестов

**Действия:**

- Создать отдельный Docker Compose файл для тестовой БД
- Настроить автоматический запуск/остановку контейнеров
- Настроить миграции БД перед запуском тестов
- Реализовать seed данных для тестов

### 3. Реализация Page Object Model

**Файлы:**

- `e2e/page-objects/BasePage.ts` - базовый класс для всех страниц
- Все страницы из структуры выше

**Действия:**

- Создать базовый класс с общими методами (wait, navigation)
- Реализовать Page Objects для всех основных страниц
- Добавить методы для взаимодействия с элементами
- Реализовать проверки состояний страниц

### 4. Тестовые фикстуры и утилиты

**Файлы:**

- `e2e/fixtures/auth.ts` - авторизация пользователей
- `e2e/fixtures/db.ts` - работа с тестовой БД через Prisma

**Действия:**

## Технические детали

### Конфигурация Playwright

```typescript
// playwright.config.ts
- Браузер: ТОЛЬКО Chromium (Firefox и WebKit не используются)
- Установка: npx playwright install chromium
- Режим: headed/headless
- Timeout: 30s для действий, 60s для навигации
- Скриншоты: только при падении
- Видео: только при падении
- Retry: 2 попытки для flaky тестов
- Workers: 4 параллельных воркера
```

**Важно:** В конфигурации `playwright.config.ts` должен быть указан только один проект с браузером `chromium`. Другие браузеры (firefox, webkit) не должны быть в конфигурации.

### Тестовая база данных

- Отдельная схема БД для тестов
- Автоматическая очистка перед каждым тестом (опционально)
- Seed данных через Prisma
- Использование транзакций для изоляции тестов

### Селекторы

- Приоритет: `data-testid` атрибуты
- Резерв: семантические селекторы (role, text)
- Избегать: CSS селекторы, XPath

## Метрики успеха

- Покрытие: все критические пользовательские сценарии
- Стабильность: >95% успешных прогонов
- Скорость: полный прогон <10 минут
- Поддержка: легко добавлять новые тесты

## Риски и митигация

1. **Flaky тесты**: Использовать retry, стабильные селекторы, явные ожидания
2. **Медленные тесты**: Параллельный запуск, оптимизация ожиданий
3. **Поддержка тестов**: Документация, примеры, code review
4. **Тестовая БД**: Изоляция через Docker, автоматическая очистка
