# Исправление бага Stage5: Ошибка на последнем слове

## 🐛 Проблема

**Этап:** Stage5 (Составление предложения)

**Баг:** Если на последнем слове/фразе допустить ошибку, этап завершается успешно вместо того чтобы показать слово в retry mode.

**Воспроизведение:**

1. Дойти до последнего слова/предложения на Stage5
2. Все предыдущие выполнены правильно
3. Допустить ошибку в последнем предложении
4. **Результат:** Этап завершается успешно ❌
5. **Ожидалось:** Переход в retry mode для повтора ошибочного слова ✅

## 🔍 Причина

**Race condition** между обновлением состояния и его проверкой:

### Было (баг):

```typescript
// В completePhrase():
setExerciseResults(prev => {
    const newResults = [...prev];
    newResults[exerciseIndex] = correct;
    return newResults;
});

// Затем через setTimeout:
setTimeout(() => {
    // ...
    handleNext(); // ← Использует СТАРОЕ exerciseResults из замыкания!
}, delay);

// В handleNext():
const errorIndices = exerciseResults // ← СТАРОЕ состояние!
    .map((result, idx) => (result === false ? idx : -1))
    .filter(idx => idx !== -1);
```

**Проблема:**

1. `setExerciseResults` обновляет состояние асинхронно
2. `handleNext` вызывается из setTimeout и использует `exerciseResults` из замыкания
3. На момент проверки `exerciseResults` еще не содержит последнюю ошибку
4. `errorIndices.length === 0` → этап завершается ❌

### Стало (исправлено):

```typescript
setTimeout(() => {
    // ...
    if (isRetryMode) {
        // retry mode логика
    } else {
        // Обычный режим
        const currentWordPhrases = wordPhrases[currentIndex] || [];

        if (currentPhraseIndex < currentWordPhrases.length - 1) {
            setCurrentPhraseIndex(currentPhraseIndex + 1);
        } else if (currentIndex < wordsWithPhrases.length - 1) {
            setCurrentPhraseIndex(0);
            setCurrentIndex(currentIndex + 1);
        } else {
            // ✅ Для последнего слова используем callback с актуальным состоянием
            setExerciseResults(currentResults => {
                setHasCompletedFirstRound(true);

                // Проверяем АКТУАЛЬНОЕ состояние
                const errorIndices = currentResults
                    .map((result, idx) => (result === false ? idx : -1))
                    .filter(idx => idx !== -1);

                if (errorIndices.length > 0) {
                    // Есть ошибки - переходим в retry mode ✅
                    setIsRetryMode(true);
                    // ... переход к первой ошибке
                } else {
                    // Все правильно - завершаем этап
                    onComplete();
                }

                return currentResults;
            });
        }
    }
}, delay);
```

## ✅ Решение

**Ключевые изменения:**

1. **Убран вызов `handleNext()` в обычном режиме**
    - Логика перехода вынесена прямо в timeout callback

2. **Для последнего слова используется callback**

    ```typescript
    setExerciseResults(currentResults => {
        // Работаем с АКТУАЛЬНЫМ состоянием
        const errorIndices = currentResults.map(...);
        // ...
        return currentResults;
    });
    ```

3. **Гарантия актуальности данных**
    - `currentResults` в callback - это всегда актуальное состояние
    - Проверка ошибок происходит с учетом последнего обновления

## 📝 Технические детали

### Почему это работает?

**React setState с callback:**

```typescript
setExerciseResults(currentResults => {
    // currentResults - это АКТУАЛЬНОЕ состояние на момент выполнения
    // даже если setState был вызван несколько раз
    return currentResults;
});
```

| Stage  | Проблема была? | Почему                                                   |
| ------ | -------------- | -------------------------------------------------------- |

| Stage4 | ❌ Нет         | Аналогично Stage2                                        |

| Stage6 | ❌ Нет         | Простая структура, один уровень                          |

### Stage5 - особый случай:

Stage5 имеет двухуровневую структуру:

- **Слова** (words) → каждое слово может иметь несколько
- **Фраз** (phrases) → каждая фраза - отдельное упражнение

Из-за этого логика перехода сложнее:

1. Проверка: есть ли еще фразы у текущего слова
2. Проверка: есть ли еще слова
3. Только если оба условия false → проверка на ошибки
